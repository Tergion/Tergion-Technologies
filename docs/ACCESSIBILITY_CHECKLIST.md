# Accessibility Checklist

## Structure

- Use semantic HTML.
- Keep headings in a logical order.
- Ensure navigation is keyboard accessible.
- Provide a skip link for main content.
- Do not hide essential content behind animation or hover-only states.

## Focus And Interaction

- Provide visible focus states.
- Ensure dialogs can be opened, used, and closed with a keyboard.
- Keep mobile bottom navigation reachable and understandable.
- Use buttons for actions and links for navigation.

## Forms

- Tie labels to form fields.
- Tie validation messages to the relevant fields.
- Keep required fields clear.
- Keep optional fields clearly optional.
- Do not require SMS consent for request submission.
- Use generic but helpful error messages.

## Visual Design

- Maintain readable color contrast.
- Avoid tiny low-contrast text.
- Respect reduced-motion preferences.
- Do not communicate essential meaning only through color, icons, or animation.
- Check mobile layouts at narrow widths for clipping, overlap, and blocked content.
